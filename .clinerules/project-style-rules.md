## Правила работы с CSS в проекте SochiFest2026

### Общие правила CSS
1. **Всегда проверять CSS на наличие сбросов стилей** - в проекте есть глобальный сброс `#inclusive-dance-landing-2026 * { border-width: 0 !important; }`
2. **Учитывать важность селекторов** - использовать `!important` для переопределения сбросов
3. **Проверять конфликты стилей** - анализировать существующие стили перед внесением изменений
4. **Учитывать специфичность селекторов** - более специфичные селекторы имеют приоритет

### Специфика проекта
1. **Bitrix интеграция** - все стили изолированы в контейнере `#inclusive-dance-landing-2026`
2. **Сброс бордюров** - все элементы имеют `border-width: 0 !important`
3. **Tailwind CSS** - фреймворк используется с префиксом `!important`
4. **Пользовательские цвета** - определены в `tailwind.config`

### Практические рекомендации
1. **Для бордюров** - использовать CSS-классы с `!important` вместо inline-стилей
2. **Тестировать изменения** - проверять визуально, если стили не применяются
3. **Документировать** - записывать найденные особенности в правила проекта

### Найденные проблемы и решения
- Проблема: бордюры не отображаются из-за сброса
- Решение: добавить специальный CSS-класс с `!important` в блок `<style>`
